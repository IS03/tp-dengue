REQUISITOS DEL SISTEMA DE PROCESAMIENTO Y NORMALIZACIÓN DE DATOS DE DENGUE EN ARGENTINA
=======================================================================================

DESCRIPCIÓN GENERAL
===================
Sistema completo de procesamiento, normalización y análisis de datos epidemiológicos de dengue en Argentina. 
El sistema integra datos de casos de dengue con información demográfica y geográfica para crear un dataset 
unificado y normalizado que permite análisis epidemiológicos detallados.

OBJETIVOS PRINCIPALES
=====================
1. Normalización de datos: Estandarizar formatos, nombres y códigos de identificación
2. Integración de fuentes: Combinar datos de dengue con información demográfica y geográfica
3. Calidad de datos: Implementar verificaciones y correcciones automáticas
4. Trazabilidad: Mantener backups y registros de todos los cambios realizados
5. Análisis epidemiológico: Facilitar estudios de incidencia, distribución temporal y espacial

FUENTES DE DATOS
================

1. DATOS DE DENGUE
------------------
- Fuente: Ministerio de Salud de Argentina
- Período: 2018-2025
- Formato original: Excel/CSV con información semanal
- Variables: Casos por departamento, provincia, grupo etario, semana epidemiológica
- Ubicación: dataset-dengue/bruto/
- Archivos procesados: dataset-dengue/procesado/

2. DATOS DEMOGRÁFICOS
---------------------
- Fuente: INDEC (Instituto Nacional de Estadística y Censos)
- Período: 2018-2025
- Cobertura: Todos los departamentos de Argentina
- Variables: Población por departamento y año
- Ubicación: dataset-poblacion/bruto/
- Archivos procesados: dataset-poblacion/procesado/

3. DATOS GEOGRÁFICOS
--------------------
- Fuente: Lista oficial de departamentos
- Cobertura: 531 departamentos de Argentina
- Variables: Códigos UTA 2020, coordenadas, información socioeconómica
- Ubicación: dataset-departamentos/bruto/
- Archivo procesado: dataset-departamentos/procesado/lista-departamentos.csv

FLUJO DE PROCESAMIENTO
======================

FASE 1: NORMALIZACIÓN BÁSICA
-----------------------------
1. Conversión de formatos: Excel → CSV
2. Limpieza de texto: Eliminación de acentos, caracteres especiales
3. Estandarización de columnas: Nombres consistentes
4. Eliminación de datos vacíos: Filas y columnas sin información

Scripts principales:
- normalizar_dengue.py: Normalización básica de archivos de dengue
- limpiar_csv.py: Limpieza general de archivos CSV
- normalizar_texto.py: Normalización de texto

FASE 2: NORMALIZACIÓN DE DATOS DE DENGUE
----------------------------------------
1. Normalización de departamentos y provincias:
   - Verificación contra lista oficial
   - Corrección de nombres inconsistentes
   - Asignación de códigos UTA 2020

2. Normalización de grupos etarios:
   - Estandarización de rangos de edad
   - Corrección de inconsistencias temporales

3. Manejo de datos faltantes:
   - Identificación de valores desconocidos
   - Corrección automática e interactiva

Scripts principales:
- verificar_departamentos_provincias.py: Verificación de consistencia geográfica
- normalizar_grupos_edad.py: Estandarización de rangos etarios
- corregir_problemas_automatico.py: Corrección automática de problemas

FASE 3: NORMALIZACIÓN DE DATOS DEMOGRÁFICOS
-------------------------------------------
1. Verificación de partidos:
   - Validación contra lista oficial
   - Corrección de nombres incorrectos

2. Verificación de valores:
   - Detección de valores nulos o incorrectos
   - Conversión a enteros

3. Asignación de códigos UTA:
   - Agregación de identificadores únicos

Scripts principales:
- verificar_y_corregir_partidos.py: Validación de nombres de partidos
- verificar_valores_poblacion.py: Detección de valores incorrectos
- agregar_uta_id_poblacion.py: Asignación de códigos UTA

FASE 4: INTEGRACIÓN DE DATOS
-----------------------------
1. Merge con población:
   - Agregación de datos demográficos por departamento y año
   - Matching: provincia + departamento + año

2. Consolidación final:
   - Unificación de todos los datasets
   - Verificación de consistencia

Scripts principales:
- agregar_poblacion_dengue.py: Merge de datos demográficos con dengue
- crear_columna_uta_id.py: Asignación de códigos UTA 2020

DATASET FINAL
=============

Estructura del Dataset Consolidado (dengue-final.csv):
- Registros: 68,127 filas
- Período: 2018-2025
- Departamentos: 531 departamentos de Argentina
- Cobertura: 24 provincias + CABA
- Variables: 14 columnas normalizadas

Columnas del dataset final:
1. departamento_nombre: Nombre del departamento
2. provincia_id: ID numérico de la provincia
3. provincia_nombre: Nombre de la provincia
4. ano: Año de los datos
5. semanas_epidemiologicas: Semana epidemiológica
6. evento_nombre: Tipo de evento (dengue)
7. grupo_edad_id: ID del grupo etario
8. grupo_edad_desc: Descripción del grupo etario
9. cantidad_casos: Número de casos
10. departamento_nombre_normalizado: Nombre normalizado
11. provincia_nombre_normalizado: Provincia normalizada
12. fila_original: Referencia al archivo original
13. departamento_id_uta_2020: Código UTA 2020
14. poblacion: Población del departamento
